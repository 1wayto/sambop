# Sambop Git PR Workflow

## Overview

Sambop keeps GitHub as the source of truth. Humans and AI agents contribute through the same workflow: clone the repository, create a focused branch, make reviewable changes, push, and open a pull request.

## Steps

1. Read `README.md`, `SAMBOP.md`, and `AGENTS.md`.
2. Create a focused branch.
3. Make a small change.
4. Verify the change.
5. Commit with a clear message.
6. Push the branch.
7. Open a pull request.
8. Include evidence and risk notes.

## Pitfalls

- Do not push directly to the default branch.
- Do not make broad unrelated changes.
- Do not omit verification.
- Do not submit fake evidence or generated noise.

## Verification

A valid Sambop contribution has a PR, a clear explanation, and enough evidence for maintainers to review it.
